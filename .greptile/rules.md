# 代码审查规则

## TypeScript/JavaScript

- 使用明确的类型定义，避免 `any` 类型
- 异步操作必须正确处理错误
- 使用 `const` 优先，必要时才用 `let`
- 避免嵌套过深的回调，使用 async/await

## 安全性

- 不直接拼接用户输入到 SQL 查询
- 不使用 `eval()` 或 `Function()` 处理用户输入
- 敏感信息不记录到日志
- 验证所有外部输入

## 性能

- 避免在循环中进行数据库查询（N+1 问题）
- 大列表使用虚拟滚动或分页
- 避免不必要的重渲染

## 测试

- 新功能需要添加测试
- 边界条件需要测试覆盖
- 错误路径需要测试

## 文档

- 公共 API 需要注释说明
- 复杂逻辑需要解释
- 保持 README 更新

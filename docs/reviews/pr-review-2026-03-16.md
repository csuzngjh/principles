## 3) 详细问题清单

## 🔴 P1 - `--features` 参数未生效（功能缺失）

**现象**

- `index.ts` 已解析 `--features`，并将其注入 `installOptions.features`。
- 但 `installer.ts` 的核心流程（复制 Skills / Core Templates / Principles Layer）没有根据 `options.features` 进行任何筛选。

**影响**

- 自动化系统传入 `--features reflection` 这种“最小安装”期望不会被满足。
- 文档中“可按模块安装”的承诺失真。
- 会增加不必要的模板注入和认知噪音，影响 LLM 提示词体积与稳定性。

**建议修复**

1. 在安装流程中引入 feature gate（最小可行）：
   - 将模板与能力映射到 feature key；
   - 在复制阶段按 `options.features` 过滤；
   - 默认值保持兼容（`evolution,trust,pain`）。
2. 增加非法 feature 校验：
   - 未识别项 warning + 忽略（或直接 fail-fast）。
3. 增加 e2e / 单元测试：
   - `--features reflection` 时不应安装 trust/evolution 对应模板。

## 🟡 P2 - 文档默认值与实现表达易混淆

文档中写法会让读者认为 `--smart` 默认开启；实现上实际是：

- 当进入非交互模式时，若未显式指定 `--force`，代码会默认 `mode = smart`。
- 但 CLI 层 `--smart` 本身默认值仍为 false。

建议在文档中明确：

- “`--smart` 是显式开关；非交互模式下如果未指定 `--force`，会自动使用 smart 模式。”

## 4) 输入给 LLM 的“最终完整提示词”是怎样的

当前 `before_prompt_build` Hook 的拼装顺序（高到低）如下：

1. `prependSystemContext`
   - `<core_principles>...</core_principles>`（来自 PRINCIPLES）
   - `<thinking_os>...</thinking_os>`（来自 THINKING_OS）

2. `prependContext`
   - （可选，最高优先）`[🚨 SYSTEM OVERRIDE 🚨] ...`（当 EVOLUTION_QUEUE 有 in_progress）
   - `<pd:internal_context>...</pd:internal_context>`（trust score / hygiene 等动态内容）
   - `<reflection_log>...</reflection_log>`
   - `<project_context>...</project_context>`（非 minimal mode）
   - `<system_capabilities>...</system_capabilities>`（非 minimal mode）
   - `<heartbeat_checklist>...</heartbeat_checklist>`（heartbeat 触发）

3. `appendSystemContext`
   - 当前为空字符串（保留扩展位）。

同时存在 size guard：总长度超过 10000 字符时，优先截断 `<project_context>` 到前 50 行。

## 5) 可直接落地的 Prompt 优化建议

以下优化目标：**更稳、更短、更可控**。

1. **统一“指令层级协议”**
   - 建议把硬指令统一为结构化块（而非自然语言段落）：
     - `role=policy`, `priority=critical`, `scope=single_turn`。
   - 降低模型对“系统覆盖文本”理解歧义。

2. **把动态态势信息压缩成 key-value**
   - 目前 `<pd:internal_context>`是自然语言 + 标记混排。
   - 改为：
     ```text
     trust_score=65
     trust_stage=2
     persists_today=1
     advisory=none
     ```
   - 能显著减 token，减少模型误读。

3. **将 evolution override 改成“机器可解析协议”**
   - 现在是长段英文说明 + 代码块。
   - 可改为：
     ```text
     [PD_TASK]
     type=evolution
     action=sessions_spawn
     target=diagnostician
     model=...
     message=...
     ack=[EVOLUTION_ACK]
     ```
   - 更短且更稳定。

4. **size guard 升级为分级裁剪**
   - 目前只裁 project_context。
   - 建议顺序：
     1) heartbeat_checklist（保留摘要）
     2) reflection_log（保留最近 N 条）
     3) system_capabilities（保留能力索引）
     4) project_context（最后裁）

5. **显式标记 minimal mode 行为**
   - 在 prompt 中加入一行 machine flag：`mode=minimal`，帮助模型理解为何缺少部分上下文。

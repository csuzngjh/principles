## 2. 模块一：动态痛觉词典 (Track B: Dynamic Pain Dictionary)
*定位：用于兜底捕捉 AI 在自然语言输出中流露出的主观困惑与逻辑死循环。*

### 2.1 存储结构 (`pain_dictionary.json`)
存放于 `stateDir` 目录下，支持热更新。

```json
{
  "rules": {
    "P_CONFUSION_ZH": {
      "type": "regex",
      "pattern": "我(似乎|好像)(不确定|不太确定|不清楚|困惑)",
      "severity": 35,
      "hits": 0,
      "status": "active"
    },
    "P_LOOP_EN": {
      "type": "exact_match",
      "phrases": ["going in circles", "back to square one"],
      "severity": 45,
      "hits": 0,
      "status": "active"
    }
  }
}
```

### 2.2 运行策略
1.  **内存加速**：插件启动时将词典载入内存 Map，并预编译所有正则。
2.  **命中统计**：`llm_output` 钩子在内存中进行原子匹配，实时更新 `hits`。
3.  **异步同步**：由 `EvolutionWorker` 每 15 分钟将内存数据同步至硬盘，避免同步 I/O 带来的 `FailoverError`。

## 3. 模块二：通用阻力指数引擎 (Track A: Generalized Friction Index)
*定位：物理级监控，通过拦截工具执行结果，强制中断“自信且愚蠢”的错误行为。*

### 3.1 阻力计算公式 (The GFI Formula)
每次工具调用后计算：
**GFI_new = max(0, GFI_old + (Delta_F * M_loop)) - C_cool**

*   **Delta_F (基础阻力)**：
    *   Exit Code != 0: +30
    *   Syntax Error / JSON Error: +40
    *   User Intervene (用户否定): +80
*   **M_loop (循环乘数)**：基于去噪哈希（移除时间戳/内存地址）对比。若连续犯同类错误，阻力按 `1.5^n` 放大。
*   **C_cool (冷却重置)**：一旦工具执行完全成功，阻力指数立即清零。

### 3.2 强力干预机制
当 **GFI > 100** 时，`before_prompt_build` 钩子执行：
1.  **物理锁死**：暂时拦截后续工具调用权限。
2.  **系统置顶**：在 Prompt 最上方注入强制指令，要求 AI 必须进入 `<reflection>` 模式。

## Phase 4 — 核心门禁与自动检查脚本（2天）

> 目标：让系统“即使主智能体忘了流程，也会被门禁推回正确轨道”。

### 4.1 创建 hooks 脚本目录
- [ ] 创建 `.claude/hooks/`

### 4.2 写 4 个脚本（按模板粘贴）
- [ ] `pre_write_gate.sh`：写入门禁（risk_paths + PLAN/AUDIT PASS 才允许写）
- [ ] `post_write_checks.sh`：写入后自动跑检查（失败写 pain flag）
- [ ] `stop_evolution_update.sh`：Stop 时落 ISSUE_LOG（仅在 pain flag 存在时）
- [ ] `precompact_checkpoint.sh`：PreCompact 时写 CHECKPOINT

### 4.3 PROFILE.json 配置（务必可运行）
- [ ] 在 `docs/PROFILE.json` 配置 tests.commands
  - [ ] 把 `npm test` 等命令改成你项目能跑通的最小命令（建议 smoke 很快）
  - [ ] 确保 `jq` 可解析

### 4.4 关键：设置风险路径
- [ ] 在 `docs/PROFILE.json` 中设置 `risk_paths`
  - 建议至少包含一个你能测试的目录（比如 `src/server/` 或 `infra/`）

验收 A（门禁）：
- [ ] 删除或临时移走 `docs/PLAN.md`，尝试让 Claude 修改 risk_paths 内文件
  - 期望：被 `pre_write_gate.sh` 阻断（exit 2），stderr 提示缺 PLAN/AUDIT
- [ ] 恢复 `docs/PLAN.md`，但让 `docs/AUDIT.md` 的 `RESULT:` 不是 PASS
  - 期望：仍被阻断
- [ ] 把 `docs/AUDIT.md` 改成 `RESULT: PASS`
  - 期望：允许写入

验收 B（自动检查 + 痛苦落盘）：
- [ ] 故意做一个会让测试失败的小改动（或改 tests 命令返回非 0）
  - 期望：`post_write_checks.sh` 写入 `docs/.pain_flag`
- [ ] 结束回合（让 Claude 输出完成）
  - 期望：`stop_evolution_update.sh` 追加 `docs/ISSUE_LOG.md` 新条目，并删除 `.pain_flag`

验收 C（PreCompact）：
- [ ] 手动触发 `/compact`（或让对话足够长触发自动 compact）
  - 期望：`docs/CHECKPOINT.md` 新增一条 checkpoint

## 常见故障排查（初级同学必看）

1) **hooks 不触发**
- 检查 `.claude/settings.json` 路径是否正确
- 确认脚本 `chmod +x` 了
- 脚本首行要有 `#!/usr/bin/env bash`

2) **门禁一直阻断**
- 检查 `docs/AUDIT.md` 是否包含精确行：`RESULT: PASS`
- 检查 `docs/PLAN.md` 是否存在（或 require_plan 为 true）

3) **自动检查没跑**
- 检查 `docs/PROFILE.json` 的 `tests.commands.smoke/unit/full` 是否为空
- 手动在项目根目录运行命令确认能跑通

4) **ISSUE_LOG 没新增**
- 检查 `docs/.pain_flag` 是否真的生成
- stop 脚本是否能写入（文件权限/路径）

#### 情况 1：要修改危险文件

```bash
claude> 帮我修改 infra/config.yaml

🤔 系统：等等！这个文件很危险
❌ Blocked: risk edit requires docs/PLAN.md

📝 你需要做什么：
1. 告诉 Claude："进入 Plan Mode"
2. Claude 会自动制定计划
3. 审核计划没问题后
4. Claude 自动执行
```

#### 情况 2：上次任务失败了

```bash
claude> 帮我修复登录 bug

⚠️ 系统启动时显示：
⚠️ 检测到未处理的 pain flag
⚠️ 建议：先解决上次的失败

📝 你需要做什么：
1. 告诉 Claude："查看 pain flag"
2. Claude 会显示上次失败的原因
3. 决定是继续还是放弃
4. 解决后自动清理
```

#### 情况 3：测试没通过

```bash
claude> 修改代码后...

❌ Post-write checks failed
❌ Pain flag written to docs/.pain_flag

📝 你需要做什么：
1. 告诉 Claude："查看测试失败原因"
2. Claude 会修复问题
3. 测试通过后自动清理 pain flag
```

## 🔧 配置（一次设置，永久生效）

### 基础配置（推荐）

编辑 `docs/PROFILE.json`：

```json
{
  "risk_paths": ["infra/", "db/", "prod/"],
  "gate": {
    "require_plan_for_risk_paths": true
  }
}
```

**作用**：告诉系统哪些目录是危险的

**你需要做的**：
1. 把你觉得危险的目录加到 `risk_paths`
2. 保存文件
3. 完成！系统会自动保护这些目录

### 高级配置（可选）

```json
{
  "tests": {
    "commands": {
      "smoke": "npm test --silent"
    }
  }
}
```

**作用**：修改文件后自动运行测试

**你需要做的**：改个命令，其他自动。

## ❓ 常见问题（FAQ）

### Q1: 我需要理解系统怎么工作吗？

**A**: 不需要！你就像平时一样用 Claude，系统会在后台保护你。

### Q2: 我需要手动运行什么脚本吗？

**A**: 不需要！所有自动化脚本都由 Claude Code 自动触发。

### Q3: 我需要查看那些日志文件吗？

**A**: 不需要！除非你想排查问题。系统会自动管理日志。

### Q4: 系统会不会太烦人？

**A**: 不会！
- ✅ 90% 的情况：系统静默工作
- ⚠️ 10% 的情况：系统会提示你
- ❌ 1% 的情况：系统会阻止你（危险操作）

### Q5: 我能绕过系统的检查吗？

**A**:
- 普通文件：可以，系统不检查
- 危险文件：不能，必须经过流程（这是保护你）

### Q6: 系统失败怎么办？

**A**:
```bash
# 系统会留下 pain flag
# 下次启动时会提醒你
# 你只需要说："解决它"
# Claude 会自动处理
```

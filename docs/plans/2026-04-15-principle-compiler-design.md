# Principle Compiler Design: Unified Reflection Pipeline

**Date**: 2026-04-15
**Status**: Draft
**Core Goal**: Turn principles into enforceable rules, not soft context suggestions.

## 1. Problem

PD has 74 principles but zero active code implementations. RuleHost is functional code but returns passthrough because `_tree.implementations` is empty. The system can reflect but cannot enforce.

Current state of the chain:

```
Pain → Diagnostician → Principle (JSON) → ??? → RuleHost.evaluate() → gate.ts block
                                        ↑
                                    THIS IS BROKEN
```

## 2. Architecture: Unified Reflection Pipeline

### 2.1 Core Insight

Three existing subsystems share the same input: Principle + Pain + Session. They differ only in output:

| Subsystem | Input | Output |
|-----------|-------|--------|
| Nocturnal (LoRA training) | Principle + Pain + Session | ORPO training data |
| Principle Internalization | Principle + Rule metrics | Route recommendation |
| **Principle Compiler (new)** | Principle + Pain + Session | Executable JS + ledger registration |

Unify the input collection into one module. Each subsystem becomes a "sink" that processes the same context.

### 2.2 Pipeline Flow

```
┌─────────────────────────────┐
│  ReflectionContextCollector  │  ← one entry point
│  collect(principleId)        │
│    → principle from ledger   │
│    → pain events from DB     │
│    → session snapshot        │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│  InternalizationRouter      │  ← existing routing policy, now dispatches
│  'code' | 'skill' | 'defer' │
└──┬──────────┬──────────┬────┘
   │          │          │
   ▼          ▼          ▼
CodeSink   SkillSink   (existing Nocturnal)
```

### 2.3 Interfaces

```typescript
/** Unified input — every sink receives the same structure */
interface ReflectionContext {
  principle: Principle;
  painEvents: NocturnalPainEvent[];
  sessionSnapshot: NocturnalSessionSnapshot;
  lineage: {
    sourcePainIds: string[];
    sessionId: string;
  };
}

/** Shared collector — reuses existing NocturnalTrajectoryExtractor */
class ReflectionContextCollector {
  collect(principleId: string): ReflectionContext | null;
  collectBatch(filter?: { status?: string }): ReflectionContext[];
}

/** Each sink implements this */
interface ReflectionSink {
  readonly route: 'code' | 'skill' | 'training';
  process(ctx: ReflectionContext): Promise<SinkResult>;
}

interface SinkResult {
  success: boolean;
  artifacts: string[];
  ledgerUpdates: LedgerUpdate[];
}
```

## 3. Principle Compiler (CodeSink)

### 3.1 Compilation Strategy: Hybrid

Two paths based on pain event complexity:

| Pain Pattern | Path | How |
|---|---|---|
| Single tool + path/error match | Template | Extract regex from pain.toolName + pain.filePath |
| Multiple tools or complex conditions | LLM | Feed ReflectionContext to LLM, get evaluate() code |
| No pain events linked | Skip | Nothing to ground the code on |

### 3.2 Compilation Flow

```
ReflectionContext
  │
  ├─ 1. Extract patterns from painEvents + sessionSnapshot
  │     - tool names that caused failures
  │     - path patterns (normalized)
  │     - error types / messages
  │     - user correction cues
  │
  ├─ 2. Select path
  │     - simple pattern (1-2 tools, clear regex) → Template
  │     - complex (3+ tools, conditional logic) → LLM
  │
  ├─ 3. Generate evaluate() function
  │     - Must conform to RuleHost sandbox (no require/import/fetch)
  │     - Static validation before writing to disk
  │
  ├─ 4. Write .js file to .state/principles/implementations/
  │
  └─ 5. Register in ledger
        - Create Rule in _tree.rules (status: proposed, enforcement: block)
        - Create Implementation in _tree.implementations
        - lifecycleState: candidate (needs promotion)
```

### 3.3 Generated Code Format

```javascript
// Auto-generated by Principle Compiler
export const meta = {
  name: 'Auto_P_066',
  version: '1.0.0',
  ruleId: 'R_P066_auto',
  coversCondition: 'Prevent task logic in heartbeat files',
  compiledAt: '2026-04-15T12:00:00Z',
  sourcePrincipleId: 'P_066',
  sourcePainIds: ['pain_a1b2', 'pain_c3d4'],
};

export function evaluate(input) {
  // tool: bash → block heartbeat+task commands
  if (input.action.toolName === 'bash') {
    const cmd = input.action.paramsSummary.command || '';
    if (/heartbeat.*(--run-tasks|task)/.test(cmd)) {
      return {
        decision: 'block',
        matched: true,
        reason: '[P_066] heartbeat不应包含task逻辑',
      };
    }
  }
  // tool: write/edit → block writing task logic into heartbeat files
  if (input.action.toolName === 'write' || input.action.toolName === 'edit') {
    const p = input.action.normalizedPath || '';
    if (/heartbeat/.test(p)) {
      const content = input.action.paramsSummary.content
        || input.action.paramsSummary.new_string || '';
      if (/task|dispatch/.test(content)) {
        return {
          decision: 'block',
          matched: true,
          reason: '[P_066] 不应向heartbeat文件写入task逻辑',
        };
      }
    }
  }
  return { matched: false };
}
```

### 3.4 LLM Path: Prompt Template

When template matching is insufficient, the LLM receives the full ReflectionContext:

```
You are a principle compiler. Generate an evaluate() function for the following principle.

PRINCIPLE:
ID: {principle.id}
Text: {principle.text}
Action: {principle.action}

EVIDENCE (pain events that triggered this principle):
{for each painEvent:
  - tool={painEvent.source}, path={painEvent.filePath}, error={painEvent.reason}, score={painEvent.score}
}

SESSION CONTEXT (what happened when this principle was violated):
{for each toolCall in sessionSnapshot where outcome=failure:
  - tool={toolCall.toolName}, params={toolCall.paramsJson}, error={toolCall.errorMessage}
}

USER CORRECTIONS:
{for each correction in sessionSnapshot.userCorrections:
  - cue={correction.cue}
}

OUTPUT: A single evaluate(input: RuleHostInput) function.
- Use only pure JavaScript (no require, import, fetch, or external APIs).
- Return { decision: 'block'|'allow'|'requireApproval', matched: boolean, reason: string }.
- The function runs in a sandbox with access only to `input`.
```

### 3.5 Static Validation

All generated code (template or LLM) must pass before writing:

1. **Parse check**: `new Function(code)` must not throw SyntaxError
2. **Forbidden patterns**: no `require`, `import`, `fetch`, `eval`, `Function`, `process`, `global`
3. **Export check**: must export `meta` object and `evaluate` function
4. **Return type check**: `evaluate(mockInput)` must return object with `matched` boolean

### 3.6 Lifecycle: Candidate → Active

New implementations enter as `candidate`. Promotion paths:

1. **Auto-promotion**: Replay engine runs the new rule against historical pain events from the same session. If it correctly blocks ≥80% of known violations with ≤20% false positives on unrelated events → auto-promote to `active`.
2. **Manual promotion**: `pd-promote-impl` command (existing).

## 6. Success Criteria

1. After `npm run compile-principles`, `.state/principles/implementations/` contains .js files for eligible principles
2. `_tree.rules` and `_tree.implementations` are populated with real entries
3. When agent violates P_066, `gate.ts` returns `block` with `source: 'rule-host'`
4. No duplicate implementation: Nocturnal, Compiler, and Skill optimization share the same collector
5. Generated code passes static validation 100% of the time (reject on failure, never write broken code)

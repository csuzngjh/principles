# Milestone v2.6: M7 Principle Candidate Intake

**Status:** ✅ SHIPPED 2026-04-27
**Phases:** m7-01 ~ m7-05
**Total Plans:** 9

## Overview

M7 consumed `principle_candidates.status=pending` candidates and made them visible/consumable via `pd candidate intake`, turning them into principle ledger entries — without triggering pain signals, deleting legacy paths, or resurrecting heartbeat/cron/subagent.

**Pipeline:** `task → run(openclaw-cli) → DiagnosticianOutputV1 → commit → artifact → principle_candidate(status=pending)` → **[M7 intake]** → `principle_ledger(status=probation)`

## Key Decisions

1. **Local Ledger Adapter**: pd-cli uses local `PrincipleTreeLedgerAdapter` (not `@principles/openclaw-plugin`) — CLI must work without openclaw-plugin SDK dependency
2. **sourceRef idempotency key**: Uses `candidate://<candidateId>` — avoids one-artifact→N-candidates collision
3. **existsForCandidate matching**: By `sourceRef === 'candidate://<candidateId>'`, not by artifactId
4. **candidate_already_consumed**: Reserved for data corruption detection only; idempotent no-op returns null
5. **Package name fix**: `@principles/openclaw-plugin` (non-existent) → `principles-disciple` (actual name)

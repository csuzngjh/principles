# Milestone v2.4: Unified Commit + Principle Candidate Intake

**Status:** ✅ SHIPPED 2026-04-24
**Phase numbers:** m5-01 through m5-05 (continuing from M4)
**Total Plans:** 7/7

## Overview

M5 delivered the artifact registry + committer pipeline: diagnostician output is atomically committed to SQLite as diagnosis artifacts, from which `kind=principle` recommendations are extracted as principle candidates. Full traceability chain: task → run → output → artifact → candidate → resultRef.

## Milestone Summary

**Key Decisions:**

- SQLite-only atomic commit: all truth in `.pd/state.db`, no filesystem ledger as transaction participant
- Committer interface isolation: Runner only depends on `DiagnosticianCommitter`, never references artifact/candidate table names or SQL
- resultRef URI scheme `commit://<commitId>`: commit record is the link between task resultRef and artifact/candidates
- Description length guard: 10,000 char application-layer limit before SQLite insert to prevent resource exhaustion
- PDRuntimeError preservation: already-typed errors re-thrown unchanged through transaction catch blocks

**Issues Resolved:**

- m5-04 verification gaps (CLIV-04 JSON output, TELE-01/02/03 event timing) resolved before phase close
- Schema test FK mismatches (task_id/run_id suffix inconsistencies) auto-fixed during test execution

**Issues Deferred:** None

# v1.15 Requirements - Runtime & Truth Contract Hardening

- **RT-01**: OpenClaw-dependent runtime calls execute through an explicit adapter boundary
- **RT-02**: Workspace/session/model/provider ingress is validated before runtime execution
- **RT-03**: Runtime boundary drift is covered by contract tests
- **RT-04**: Hidden runtime fallbacks and machine-specific defaults are removed from the production path
- **TRUTH-01**: Exports and datasets only assert evidence-backed facts
- **TRUTH-02**: Missing evidence remains `unknown`, `not_observed`, or omitted
- **TRUTH-03**: Zero-evidence and mixed-evidence cases are test-covered
- **OBS-01**: Key production invariants are machine-checkable
- **OBS-02**: Diagnostics distinguish contract failure classes
- **MERGE-01**: Merge-gate fixes are separated from deferred hardening work
- **MERGE-02**: Focused end-to-end verification demonstrates real state transitions before merge recommendation

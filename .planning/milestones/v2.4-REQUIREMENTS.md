# Requirements Archive: v2.4 — Unified Commit + Principle Candidate Intake

**Archived:** 2026-04-24
**Milestone:** v2.4 M5 Unified Commit + Principle Candidate Intake
**Core Value:** diagnostician output -> diagnosis artifact -> principle candidate -> task resultRef，全链路在 SQLite .pd/state.db 内原子完成

---

## Artifact Registry

- [x] **ARTF-01**: artifacts 表 schema — id, task_id, run_id, artifact_kind, content_json, created_at
- [x] **ARTF-02**: commits 表 schema — id, task_id, run_id, artifact_id, idempotency_key, status, created_at
- [x] **ARF-03**: principle_candidates 表 schema — id, artifact_id, task_id, source_run_id, title, description, confidence, source_recommendation_json, idempotency_key, status, created_at
- [x] **ARTF-04**: 8 indexes across 3 tables + CASCADE FK chains

## DiagnosticianCommitter

- [x] **COMT-01**: DiagnosticianCommitter interface defined
- [x] **COMT-02**: Transaction-wrapped commit (artifact + commit + candidates in single SQLite transaction)
- [x] **COMT-03**: Extract kind='principle' recommendations as candidates
- [x] **COMT-04**: Idempotent re-commit via UNIQUE constraints on run_id and idempotency_key
- [x] **COMT-05**: Commit failure returns PDRuntimeError{artifact_commit_failed} with no orphaned rows
- [x] **COMT-06**: CommitResult returns {commitId, artifactId, candidateCount}

## Runner Integration

- [x] **RUNR-01**: DiagnosticianRunner.succeedTask() calls committer.commit() before marking task succeeded
- [x] **RUNR-02**: Correct ordering: output validated → committer.commit() → task succeeded + resultRef updated
- [x] **RUNR-03**: Commit failure → artifact_commit_failed error category, task NOT succeeded
- [x] **RUNR-04**: Runner depends only on Committer interface (no artifact/candidate table references)
- [x] **RUNR-05**: resultRef becomes `commit://<commitId>` after successful commit

## CLI + Telemetry

- [x] **CLIV-01**: `pd candidate list --task-id <id>` returns candidates for task artifacts
- [x] **CLIV-02**: `pd candidate show <candidateId>` returns title, description, confidence, source, status
- [x] **CLIV-03**: `pd diagnose status --task-id <id> --json` includes commitId, artifactId, candidateCount
- [x] **CLIV-04**: All CLI output plain TypeScript types (no CLI framework dependency), JSON for E2E
- [x] **TELE-01**: `diagnostician_artifact_committed` event emitted after commit success
- [x] **TELE-02**: `diagnostician_artifact_commit_failed` event emitted on commit failure
- [x] **TELE-03**: `principle_candidate_registered` event emitted per kind='principle' recommendation
- [x] **TELE-04**: All events use emitTelemetry() via StoreEventEmitter
- [x] **TELE-05**: RunnerPhase.Committing observable between Validating and Completed

## E2E Verification

- [x] **E2EV-01**: Happy path E2E — full traceability chain verified
- [x] **E2EV-02**: Idempotency E2E — same task/run committed twice → one artifact, no duplicate candidates
- [x] **E2EV-03**: Commit failure E2E — failure rolls back, task not succeeded
- [x] **E2EV-04**: CLI visibility E2E — pd candidate list/show resolves through resultRef chain

# Milestone v1.5: Nocturnal Helper 重构

**Status:** ✅ SHIPPED 2026-04-06
**Phases:** 6-10
**Total Plans:** 7

## Overview

Migrate Nocturnal's TrinityRuntimeAdapter to WorkflowManager interface. Built unified subagent lifecycle for all helper workflows (Empathy, DeepReflect, Nocturnal). Added WorkflowStore event tracking for Trinity 3-stage chain (Dreamer → Philosopher → Scribe). Surface degrade checks and idempotency guarantees.

## Milestone Summary

**Key Decisions:**

- Decision: Use WorkflowManager interface for NocturnalWorkflowManager (Rationale: Unified subagent lifecycle)
- Decision: Stub fallback degrades to stub implementations, NOT EmpathyObserver/DeepReflect (Rationale: NOC-15)
- Decision: Direct stage-by-stage invocation instead of runTrinityAsync for Phase 8 (Rationale: Fine-grained persistence control)
- Decision: Idempotency keys for Trinity stages (Rationale: NOC-12)
- Decision: Factory pattern for tool registration (Rationale: SDK compatibility)

**Issues Resolved:**

- Fixed context overflow for large phase counts
- Resolved Phase 7/08 verification doc gaps
- NOC-15 stub parameter signatures fixed in Phase 10

**Issues Deferred:**

- Phases 07, 08 VERIFICATION.md not created (implementations exist, low priority)
- sweepExpiredWorkflows does NOT include NocturnalWorkflowManager
- Phase 8 architectural shift from Phase 7 design (runTrinityAsync → direct stage invocation)

**Technical Debt Incurred:**

- Pre-existing TypeScript errors in evolution-reducer.ts and prompt.ts
- Test infrastructure issue with better-sqlite3 database connection

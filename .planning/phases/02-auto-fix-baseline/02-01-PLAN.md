## Phase 2: Auto-fix Baseline

This phase has 3 requirements to satisfy:
- **LINT-05**: Run `eslint --fix` for auto-fixable errors
- **LINT-06**: Audit all `eslint-disable` comments - add documented reasons
- **LINT-07**: Review all `eslint --fix` diffs before commit - no blind auto-fixes

### Research-Verified Auto-Fixable Categories

The following rules have confirmed fixers (verified via actual `eslint --fix` runs):
1. `@typescript-eslint/consistent-type-imports` - splits `import { Type }` into `import type { Type }` + `import { ... }`
2. `@typescript-eslint/array-type` - converts `Array<T>` to `T[]`
3. `@typescript-eslint/prefer-destructuring` - converts `obj.x` to `{x} = obj`
4. `@typescript-eslint/prefer-regexp-exec` - converts `.match()` to `.exec()`
5. `@typescript-eslint/prefer-readonly` - adds `readonly` modifier
6. `@typescript-eslint/no-inferrable-types` - removes redundant type annotations

### NOT Auto-Fixable (Phase 3 work)
- `no-unused-vars` / `@typescript-eslint/no-unused-vars` (432 errors - no fixer)
- `no-undef` (269 errors - missing globals)
- All other categories require manual intervention

<objective>
Create the shared workspace resolution contract and switch the highest-risk production entry points to it first.

This plan must eliminate guessed workspace writes in command wrappers and the plugin's top-level hook wiring. The output is a small shared service with explicit failure semantics, plus migrated callers.
</objective>

<must_haves>
- New shared resolver contract exists in one reusable module
- The contract no longer uses `api.resolvePath('.')` as a production fallback
- `index.ts` top-level lifecycle hooks stop using `ctx.workspaceDir || api.resolvePath('.')`
- `/pd-reflect` stops hardcoding `'main'` and stops resolving outside active command context first
- Failure behavior is explicit: command returns error; hooks either skip safely or use validated required workspace
</must_haves>

<tasks>
1. Introduce `workspace-dir-service.ts` or equivalent shared module.
   It should expose a small API for:
   - resolve required workspace
   - resolve optional workspace for safe-skip callers
   - validate and explain bad workspace directories

2. Refactor `workspace-dir-validation.ts`.
   - Remove HOME fallback semantics from the primary production path
   - Keep validation helpers if still useful
   - Make any remaining "last resort" helper test-only or explicitly deprecated

3. Migrate high-risk callers.
   - `index.ts` lifecycle hooks
   - command wrappers in `index.ts`
   - `pd-reflect.ts`

4. Add tests for the shared resolver contract.
   Minimum cases:
   - accepts valid `ctx.workspaceDir`
   - rejects HOME/root/empty
   - command path errors when no valid workspace is available
   - `/pd-reflect` prefers active command workspace over hardcoded/default resolution
</tasks>

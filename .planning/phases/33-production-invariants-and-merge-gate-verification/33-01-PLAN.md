# Plan 33-01: Implement Invariant Checks and Focused End-to-End Verification

## Objective

Turn Phase 30 merge-gate criteria into executable audit checks so merge readiness is decided by machine-checkable evidence instead of log reading and manual guesswork.

## Scope

- Add a production-facing merge-gate audit that evaluates:
  - canonical pain flag path contract
  - dataset artifact integrity
  - artifact lineage integrity
  - ORPO export integrity
  - replay evidence integrity
- Add focused tests for empty, blocked, and passing audit states
- Update planning artifacts so Phase 33 has an executable trail

## Out of Scope

- New runtime adapter work
- New replay generation flows
- Operator CLI polish

## Implementation Steps

1. Add `merge-gate-audit.ts` in the core layer with `pass/block/defer` reporting.
2. Reuse existing dataset, lineage, export, and replay storage readers instead of introducing new registries.
3. Add tests that prove:
   - empty systems defer instead of pretending to pass
   - malformed replay evidence blocks
   - populated evidence surfaces pass
4. Run focused tests plus build verification.

## Verification

- `npm test -- --run tests/core/merge-gate-audit.test.ts tests/core/replay-engine.test.ts tests/core/nocturnal-export.test.ts`
- `npm run build`

```typescript
const pendingDiagTasks = getPendingDiagnosticianTasks(wctx.stateDir);
```

REPLACE with a runtime-v2 query. Since runtime-summary-service.ts is in openclaw-plugin and runtime-v2 is in principles-core, we need a pragmatic approach:

OPTION A (preferred if SqliteTaskStore is accessible): Use the runtime-v2 SqliteTaskStore to query pending diagnostician tasks.

OPTION B (if runtime-v2 not accessible): Query the task store using a direct SQLite connection to the workspace's .principles/db/task-store.db.

The key value to preserve is `pendingTasks` count. For a M8 implementation, we can:
1. Keep the field name `heartbeatDiagnosis` for API compatibility
2. Set `pendingTasks` to query the runtime-v2 task store (SqliteTaskStore) via workspace db path

The daily stats fields (tasksWrittenToday, reportsWrittenToday, etc.) track legacy heartbeat chain metrics that no longer apply. Set them to 0 with a note that they track the legacy chain.

REPLACEMENT CODE:
```typescript
// M8: Diagnostician tasks are now managed by runtime-v2 SqliteTaskStore.
// The legacy diagnostician_tasks.json file is no longer used.
// pendingTasks now reflects the runtime-v2 task store pending diagnostician tasks.

let pendingDiagnosticianTasks = 0;
try {
  // Query runtime-v2 task store for pending diagnostician tasks
  const taskStorePath = path.join(wctx.stateDir, '..', '.principles', 'db', 'task-store.db');
  if (fs.existsSync(taskStorePath)) {
    // Use SQLite to query pending tasks of kind 'diagnostician'
    const Database = await import('better-sqlite3').catch(() => null);
    if (Database) {
      const db = new Database.default(taskStorePath, { readonly: true });
      const row = db.prepare("SELECT COUNT(*) as count FROM tasks WHERE status = 'pending' AND task_kind = 'diagnostician'").get() as { count: number } | undefined;
      pendingDiagnosticianTasks = row?.count ?? 0;
      db.close();
    }
  }
} catch {
  // If runtime-v2 task store is not accessible, pendingTasks is unknown
  pendingDiagnosticianTasks = -1; // sentinel for "unknown"
}

const heartbeatDiagnosis = {
  // M8: pendingTasks now reflects runtime-v2 SqliteTaskStore pending count
  // (replaces legacy diagnostician_tasks.json read via getPendingDiagnosticianTasks)
  pendingTasks: pendingDiagnosticianTasks,
  // M8: Legacy heartbeat chain metrics are zero — the legacy chain is removed.
  // These counters tracked the old heartbeat-injection path (M1-M7).
  tasksWrittenToday: 0,
  reportsWrittenToday: 0,
  reportsMissingJsonToday: 0,
  reportsIncompleteFieldsToday: 0,
  candidatesCreatedToday: 0,
  heartbeatsInjectedToday: 0,
};
```

<objective>
Schema-check sleep-reflection snapshot ingress and related worker inputs before they enter the nocturnal pipeline.

This plan closes the "empty/default object" loophole so malformed ingress is rejected early with explicit reasons.
</objective>

<must_haves>
- sleep-reflection snapshot ingress is schema-checked before workflow launch
- malformed snapshot payloads become explicit skip/failure reasons
- worker paths stop constructing misleading default objects from missing fields
- logs and workflow states preserve the true root cause
</must_haves>

<tasks>
1. Identify the canonical snapshot ingress type used by nocturnal and sleep-reflection.

2. Introduce a shared validation function or schema guard for ingress payloads.
   Required fields must include the identifiers and stats needed to make target-selection decisions.

3. Migrate worker / workflow callers to the new contract.
   Replace "fallback empty object" behavior with explicit rejected/skip outcomes.

4. Add regression tests.
   Minimum cases:
   - malformed snapshot payload is rejected before workflow start
   - missing required fields produce explicit failure reason
   - valid payload still flows through unchanged
</tasks>

<objective>
Centralize `.pain_flag` parsing into one explicit contract and remove scattered manual readers that silently accept malformed pain data.

The output should make missing required fields fail fast instead of quietly returning partial context.
</objective>

<context>
@.planning/PROJECT.md
@.planning/ROADMAP.md
@.planning/REQUIREMENTS.md
@.planning/phases/20-critical-data-schema-validation/20-CONTEXT.md

Critical code:
- `packages/openclaw-plugin/src/core/pain.ts`
- `packages/openclaw-plugin/src/core/pain-context-extractor.ts`
- any helper that reads `.pain_flag` line-by-line
</context>

<must_haves>
- `.pain_flag` parsing exists behind one shared parser/validator
- required fields such as `time` / `session_id` / `source` are enforced explicitly
- malformed pain files do not become "empty but usable" context objects
- callers get either valid parsed data or an explicit failure/skip result
</must_haves>

<tasks>
1. Inventory all `.pain_flag` readers and pick one canonical parser entry.

2. Introduce a typed parse result for pain data.
   It must distinguish:
   - valid pain payload
   - malformed payload
   - missing file / no data

3. Migrate current readers to the shared parser.
   Remove duplicated field-name assumptions such as `timestamp:` vs `time:`.

4. Add tests.
   Minimum cases:
   - valid `.pain_flag` with current production field names parses correctly
   - malformed or missing required fields fail explicitly
   - old scattered readers no longer silently accept bad data
</tasks>

**Validation order (per Pitfall 4 of RESEARCH.md):**
1. Guard: if `typeof output !== 'object' || output === null`, return `{ valid: false, errors: ['output must be a non-null object'] }` immediately
2. Schema validation via `Value.Check(DiagnosticianOutputV1Schema, output)` -- fail-fast in standard mode, collect in verbose
3. Task identity check: `output.taskId !== taskId` -- strict equality per D-05
4. Confidence boundary check: `output.confidence < 0 || output.confidence > 1` -- D-06
5. Non-empty summary/rootCause (TypeBox minLength:1 already enforces, but explicit check for clear error messages per D-06)
6. Evidence array shape: each evidence entry has non-empty `sourceRef` and `note`
7. Recommendations array shape: each entry has valid `kind` from RecommendationKindSchema and non-empty `description`
8. Evidence sourceRef back-check (last -- most expensive):
   - Standard: format check only (non-empty string, trimmed not empty)
   - Verbose: existence check against `options.sourceRefs` array

**Error format per D-03:**
- Build aggregate summary: `"N fields invalid: ..."` where N = number of distinct failing checks
- `errors[0]` = aggregate summary
- `errors[1..N]` = per-field detail messages
- All failures use `errorCategory = 'output_invalid'` per D-04

**Fail-fast vs collect-all (D-01):**
- Standard (`options.verbose !== true`): return immediately on first error
- Verbose (`options.verbose === true`): collect all errors before returning

**Return structure (D-07):**
```typescript
return { valid: false, errors: [aggregateSummary, ...detailErrors], errorCategory: 'output_invalid' }
```

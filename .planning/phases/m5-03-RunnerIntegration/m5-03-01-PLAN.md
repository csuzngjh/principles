# Plan m5-03-01: Runner Integration — Committer Injection + Succeed Task Pipeline

## Objective

Wire DiagnosticianCommitter into DiagnosticianRunner so that every successful diagnosis is atomically committed before the task is marked succeeded. Commit failures retry, never silently succeed.

### Task 3: Modify succeedTask to commit before marking succeeded

<read_first>
- packages/principles-core/src/runtime-v2/runner/diagnostician-runner.ts (current succeedTask method)
- packages/principles-core/src/runtime-v2/store/diagnostician-committer.ts (CommitInput/CommitResult interfaces)
</read_first>

<action>
Replace the current `succeedTask` method body. The new flow:

1. **Set phase to Committing:**
   `this.phase = RunnerPhase.Committing;`

2. **Store output in run record** (existing line, unchanged):
   `await this.stateManager.updateRunOutput(ctx.runId, JSON.stringify(ctx.output));`

3. **Call committer.commit():**
   ```
   const commitResult = await this.committer.commit({
     runId: ctx.runId,
     taskId: ctx.taskId,
     output: ctx.output,
     idempotencyKey: `${ctx.taskId}:${ctx.runId}`,
   });
   ```

4. **Use commit://{commitId} as resultRef** (replace `run://${ctx.runId}`):
   `const resultRef = \`commit://${commitResult.commitId}\`;`

5. **Mark task succeeded** (existing, but with new resultRef):
   `await this.stateManager.markTaskSucceeded(ctx.taskId, resultRef);`

6. **Emit diagnostician_task_succeeded** (existing, but add commitId to payload):
   Add `commitId: commitResult.commitId` and `candidateCount: commitResult.candidateCount` to the telemetry payload.

7. **Set phase to Completed and return** (existing, unchanged).

The idempotency key is `${ctx.taskId}:${ctx.runId}` — deterministic, derived from the task/run pair. Re-runs of the same task+run produce the same key, and the committer handles deduplication via UNIQUE constraints.

**Error handling:** Do NOT wrap the commit() call in a try-catch that catches and suppresses errors. Let any error from commit() propagate to the existing `try/catch` in `run()` which calls `handleLeaseOrPhaseError`. This routes through `classifyError` → `retryOrFail` with `artifact_commit_failed` category.

**Critical:** Remove the old `const resultRef = \`run://${ctx.runId}\`;` line. The only resultRef assignment should be the new `commit://` format.
</action>

<acceptance_criteria>
- `succeedTask` sets `this.phase = RunnerPhase.Committing` before commit
- `succeedTask` calls `this.committer.commit()` with `{ runId, taskId, output, idempotencyKey }`
- `idempotencyKey` is `${ctx.taskId}:${ctx.runId}`
- resultRef is `commit://${commitResult.commitId}` (NOT `run://`)
- `markTaskSucceeded` is called AFTER `committer.commit()`
- commit errors propagate naturally to the outer try/catch in `run()`
- Telemetry payload includes `commitId` and `candidateCount`
- No `run://` resultRef anywhere in the method
</acceptance_criteria>

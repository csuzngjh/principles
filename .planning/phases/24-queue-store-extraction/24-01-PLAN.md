<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Disk -> Store.load() | Queue file contents are untrusted; may be malformed JSON, missing fields, or contain legacy schema |
| Store.save() -> Disk | Queue items passed to save() may be malformed by calling code; must be validated before write |
| External caller -> Store methods | Callers may pass invalid items to add/save/update; validation at entry prevents corruption |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Tampering | validateQueueItemForWrite | mitigate | Permissive validation checks required fields (id, taskKind, priority, source, score, reason, timestamp, status, retryCount, maxRetries) and valid status enum. Rejects with QueueValidationError. |
| T-24-02 | Tampering | Store.load() read validation | mitigate | After JSON parse + V2 migration, validate each item. Corrupted items excluded from queue, reasons collected in QueueLoadResult. File backed up before any destructive action. |
| T-24-03 | Denial of Service | withLockAsync in all public methods | mitigate | Lock uses exponential backoff with jitter (file-lock.ts L174-182), stale lock detection (30s), max retries (50). LockUnavailableError thrown on exhaustion. |
| T-24-04 | Tampering | Queue file on disk | accept | File lock prevents concurrent writes. No integrity check (checksum) on file contents -- low-value target in local dev tool, no network exposure. |
</threat_model>

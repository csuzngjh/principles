  <name>Task 1: Add health monitoring — auto-disable on consecutive failures</name>
  <files>packages/openclaw-plugin/src/core/pd-task-reconciler.ts</files>
  <instructions>
Add to pd-task-reconciler.ts:

1. **healthCheck(tasks: PDTaskSpec[], cronStore: CronStoreFile, logger?: any): PDTaskSpec[]:**
   - For each task in tasks:
     - Find matching cron job by name
     - Read job.state.consecutiveErrors
     - If consecutiveErrors >= 3 AND NOT already autoDisabled:
       - Set task.meta.autoDisabled = true
       - Set task.meta.autoDisabledAt = Date.now()
       - Set task.meta.autoDisabledReason = `consecutiveErrors=${consecutiveErrors}`
       - Log warning: `[PD:TaskManager] Auto-disabled task '${task.id}' due to ${consecutiveErrors} consecutive errors`
   - Return updated tasks array (does NOT write — caller writes via updateSyncMeta)

2. **updateSyncMeta(task: PDTaskSpec, status: 'ok' | 'error', jobId?: string, error?: string, consecutiveErrors?: number): PDTaskSpec:**
   - Update task.meta.lastSyncedAtMs = Date.now()
   - Update task.meta.lastSyncStatus = status
   - If jobId: task.meta.lastSyncedJobId = jobId
   - If error: task.meta.lastSyncError = error
   - If consecutiveErrors !== undefined: task.meta.consecutiveFailCount = consecutiveErrors
   - If status === 'error' and consecutiveErrors !== undefined: task.meta.lastFailedAtMs = Date.now()
   - If status === 'ok': task.meta.consecutiveFailCount = 0 (reset on success)
   - Return updated task

Integrate healthCheck into reconcilePDTasks() — call after reading cron store, before diff.
</instructions>
</task>

<task type="auto">
  <name>Task 3: Add manual trigger — trigger(taskId) forces one-shot execution</name>
  <files>packages/openclaw-plugin/src/core/pd-task-reconciler.ts</files>
  <instructions>
Add to pd-task-reconciler.ts:

1. **trigger(taskId: string, workspaceDir: string, options?: { force?: boolean; logger?: any }): Promise<{ ok: boolean; error?: string }>:**
   - Read tasks from pd_tasks.json
   - Find task by id
   - If not found: return { ok: false, error: `Task '${taskId}' not found` }
   - If task.meta.autoDisabled and NOT options.force: return { ok: false, error: 'Task is auto-disabled. Use force=true to override.' }
   - Read cron store
   - Find existing job by task.name
   - If found and options.force: temporarily set enabled=true and updatedAtMs=Date.now()
   - If NOT found: CREATE one-shot job (deleteAfterRun=true, enabled=true)
   - Update task.meta.lastTriggeredAtMs = Date.now()
   - Update task.meta.lastTriggerStatus = 'pending'
   - Write updated cron store
   - Write updated tasks
   - Return { ok: true }

Integrate: trigger() writes to cron store and updates pd_tasks.json meta.
</instructions>
</task>

  <name>Task 4: Add execution history — getExecutionHistory(taskId, limit?)</name>
  <files>packages/openclaw-plugin/src/core/pd-task-reconciler.ts</files>
  <instructions>
Add to pd-task-reconciler.ts:

1. **getExecutionHistory(taskId: string, workspaceDir: string, options?: { limit?: number; status?: string }): PDTaskExecutionRecord[]:**
   - Read tasks from pd_tasks.json
   - Find task by id
   - If not found: return []
   - Get task.meta.executionHistory || []
   - Filter by status if provided (options.status: 'succeeded' | 'failed')
   - Sort by startedAt descending (most recent first)
   - Limit to options.limit || 50
   - Return filtered/sorted array

2. **recordExecution(task: PDTaskSpec, runId: string, status: PDTaskExecutionRecord['status'], error?: string): PDTaskSpec:**
   - Push new record to task.meta.executionHistory = task.meta.executionHistory || []
   - record: { runId, status, startedAt: Date.now(), endedAt: Date.now(), error }
   - Keep max 100 records (trim oldest)
   - Return updated task

Note: The actual cron execution recording happens in Phase 16 (service integration). Phase 15 only provides the query interface and the record() helper.
</instructions>

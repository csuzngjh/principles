**Required test cases (one `it` per case):**

describe('kind()'):
- returns 'pi-ai' (RS-02)

describe('getCapabilities()'):
- returns correct RuntimeCapabilities shape with supportsStructuredJsonOutput=true, supportsStreaming=false, supportsCancellation=true

describe('healthCheck()'):
- returns healthy=true when apiKeyEnv exists, getModel succeeds, and complete probe returns {"ok":true}
- returns healthy=false when apiKeyEnv is missing
- returns healthy=false when getModel throws
- returns healthy=false with degraded=true when complete probe returns unexpected result

describe('startRun()'):
- calls getModel with provider and model from config
- calls complete with correct context including UserMessage with timestamp
- passes apiKey, timeoutMs, and maxRetries: 0 to complete options
- returns RunHandle with runId, runtimeKind='pi-ai', and valid ISO startedAt
- stores output in memory (fetchOutput works after startRun completes)
- creates run state before LLM call (pollRun returns 'running' if we intercept mid-call)
- stores failed run state on error (pollRun returns 'failed' with reason after startRun rejects)

describe('JSON extraction (balanced parsing)'):
- parses plain JSON from LLM response
- parses JSON wrapped in prose text ("Here is the analysis: {...} done")
- parses JSON inside code fences ("```json\n{...}\n```")
- throws output_invalid when no JSON object found in response

describe('startRun() error mapping'):
- throws PDRuntimeError('runtime_unavailable') when apiKeyEnv missing from process.env
- throws PDRuntimeError('timeout') when LLM request is aborted/timed out
- throws PDRuntimeError('output_invalid') when LLM response contains no parseable JSON
- throws PDRuntimeError('output_invalid') when parsed JSON does not match DiagnosticianOutputV1 schema
- throws PDRuntimeError('execution_failed') after retry exhaustion on network error

describe('startRun() retry logic'):
- retries transient failures up to maxRetries times
- succeeds on second attempt after first transient failure
- does not retry on PDRuntimeError (output_invalid, runtime_unavailable)

describe('pollRun()'):
- returns terminal status (succeeded/failed) for completed run
- throws PDRuntimeError('input_invalid') for unknown runId

describe('fetchOutput()'):
- returns StructuredRunOutput with runId and payload after successful run
- returns null for unknown runId (per PDRuntimeAdapter interface: Promise<StructuredRunOutput | null>)

describe('cancelRun()'):
- resolves without error

describe('fetchArtifacts()'):
- returns empty array

describe('telemetry (AD-15)'):
- emits runtime_invocation_started event on startRun
- emits runtime_invocation_succeeded on successful completion
- emits runtime_invocation_failed on failure

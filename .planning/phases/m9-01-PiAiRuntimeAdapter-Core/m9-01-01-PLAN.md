**`healthCheck(): Promise<RuntimeHealth>`** — Three-stage probe (per M6 lesson: binary-only/list-only checks are fake probes):
1. **apiKey exists:** Check `process.env[this.config.apiKeyEnv]`. If missing → `{ healthy: false, warnings: ['API key not found in env: ...'] }`.
2. **getModel valid:** Call `getModel(config.provider as KnownProvider, config.model)`. If throws → `{ healthy: false, warnings: ['getModel failed: ...'] }`.
3. **Minimal complete probe:** Call `complete(model, { messages: [{ role: 'user', content: 'Reply with {"ok":true} only.', timestamp: Date.now() }] }, { signal: AbortSignal.timeout(30_000), apiKey, maxRetries: 0 })`. Extract JSON from response text using `extractJsonObject()`. If result is not `{ ok: true }` → `{ healthy: false, degraded: true, warnings: ['probe returned unexpected result: ...'] }`.
Return `{ healthy: true, degraded: false, warnings: [], lastCheckedAt }` only if all 3 stages pass.

**`startRun(input: StartRunInput): Promise<RunHandle>`** — This is the core one-shot method:
1. Read `apiKey = process.env[this.config.apiKeyEnv]`. If missing, throw `PDRuntimeError('runtime_unavailable', ...)`.
2. Generate `runId` (crypto.randomUUID), record `startedAt = new Date().toISOString()`.
3. **Create run state in `this.runs` Map immediately** (status: 'running') — so failed runs are always trackable via pollRun/fetchOutput.
4. Create `AbortSignal.timeout(this.config.timeoutMs ?? 300_000)` — do NOT use setTimeout+AbortController.
5. Build pi-ai Context from `input.inputPayload`:
   ```typescript
   const messageContent = typeof input.inputPayload === 'string'
     ? input.inputPayload
     : JSON.stringify(input.inputPayload);
   const userMessage: UserMessage = {
     role: 'user',
     content: messageContent,
     timestamp: Date.now(),
   };
   const context: Context = { messages: [userMessage] };
   ```
6. Get model: `getModel(config.provider as KnownProvider, config.model)`
7. Emit `runtime_invocation_started` telemetry event via `this.eventEmitter.emitTelemetry(...)`.
8. **try {**
9. Call `completeWithRetry(model, context, signal, apiKey)` (private helper, see below).
10. **Guard text content:** `const textContent = response.content.find(c => c.type === 'text'); if (!textContent) throw new PDRuntimeError('output_invalid', 'No text content in LLM response');`
11. Parse response: use `extractJsonObject(textContent.text)` to handle prose-wrapped or code-fenced JSON.
12. Validate with `Value.Check(DiagnosticianOutputV1Schema, parsed)` — if fails, throw `PDRuntimeError('output_invalid', ...)`.
13. Update run state to `status: 'succeeded'`, store output.
14. Emit `runtime_invocation_succeeded` telemetry event.
15. **} catch (err) {** — Update run state to `status: 'failed'`, set `reason`. Emit `runtime_invocation_failed` telemetry. Re-throw.
16. Return `{ runId, runtimeKind: 'pi-ai', startedAt }`.

**Private `isAbortError` helper** — Broad detection for abort/timeout errors from any provider SDK:
```typescript
private isAbortError(err: unknown, signal: AbortSignal): boolean {
  if (signal.aborted) return true;
  if (err instanceof DOMException && err.name === 'AbortError') return true;
  if (err instanceof Error && (/abort/i.test(err.message) || /timeout/i.test(err.message))) return true;
  if (typeof err === 'object' && err !== null && 'name' in err && (err as any).name === 'AbortError') return true;
  return false;
}
```

**Private `extractJsonObject` helper** — Balanced-bracket JSON extraction (reuse pattern from OpenClawCliRuntimeAdapter). Handles prose-wrapped and code-fenced JSON from LLM output. Returns `null` on failure (matching reference pattern):
```typescript
private extractJsonObject(text: string): unknown | null {
  // Try code-fenced JSON first: ```json ... ``` or ``` ... ```
  const fencedMatch = text.match(/```(?:json)?\s*\n?([\s\S]*?)\n?```/);
  if (fencedMatch) {
    try { return JSON.parse(fencedMatch[1].trim()); } catch { /* fall through */ }
  }
  // Balanced-bracket scan for first top-level {...}
  let depth = 0;
  let start = -1;
  for (let i = 0; i < text.length; i++) {
    if (text[i] === '{') { if (depth === 0) start = i; depth++; }
    else if (text[i] === '}') { depth--; if (depth === 0 && start >= 0) {
      try { return JSON.parse(text.slice(start, i + 1)); } catch { start = -1; }
    }}
  }
  return null;
}
```

Usage in startRun (step 11): `const parsed = extractJsonObject(textContent.text); if (!parsed) throw new PDRuntimeError('output_invalid', 'No valid JSON found in LLM response');`

**Private `completeWithRetry` helper:**
```typescript
private async completeWithRetry(
  model: ReturnType<typeof getModel>,
  context: Context,
  signal: AbortSignal,
  apiKey: string,
): Promise<AssistantMessage> {
  const maxRetries = this.config.maxRetries ?? 2;
  const effectiveTimeoutMs = this.config.timeoutMs ?? 300_000;
  let lastError: unknown;
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await complete(model, context, {
        signal,
        apiKey,
        timeoutMs: effectiveTimeoutMs,  // pass timeoutMs to pi-ai provider for SDK-level timeout
        maxRetries: 0,  // disable pi-ai built-in retry to avoid double-retry
      });
    } catch (err) {
      lastError = err;
      if (this.isAbortError(err, signal)) {
        throw new PDRuntimeError('timeout', `LLM request timed out after ${effectiveTimeoutMs}ms`);
      }
      if (attempt < maxRetries) {
        const delay = Math.min(1000 * Math.pow(2, attempt), 30_000);
        await new Promise(r => setTimeout(r, delay));
      }
    }
  }
  throw new PDRuntimeError('execution_failed', `LLM completion failed after ${maxRetries + 1} attempts: ${lastError}`);
}
```

**`pollRun(runId: string): Promise<RunStatus>`** — Look up in `this.runs` Map. If not found, throw `PDRuntimeError('input_invalid', ...)`. Return the stored RunStatus (always terminal for one-shot).

**`cancelRun(runId: string): Promise<void>`** — Mark run as cancelled if still in progress. For one-shot, this is a no-op since startRun blocks.

**`fetchOutput(runId: string): Promise<StructuredRunOutput | null>`** — Look up in Map. Return stored output or null if not found. Do NOT throw for missing runs (return null per interface contract).

**`fetchArtifacts(runId: string): Promise<RuntimeArtifactRef[]>`** — Return `[]` (no artifacts produced by direct LLM call).

**Error mapping rules:**
- Abort/timeout error (detected via `isAbortError()`) → `PDRuntimeError('timeout', ...)`
- JSON extraction failure (extractJsonObject throws) → `PDRuntimeError('output_invalid', ...)`
- Schema validation failure (Value.Check returns false) → `PDRuntimeError('output_invalid', ...)`
- Missing apiKeyEnv → `PDRuntimeError('runtime_unavailable', ...)`
- Retries exhausted → `PDRuntimeError('execution_failed', ...)`

**Telemetry events (AD-15):**
- `runtime_invocation_started` — emitted at start of startRun, with payload `{ runId, runtimeKind: 'pi-ai', provider, model, timeoutMs }`
- `runtime_invocation_succeeded` — emitted after successful validation, with payload `{ runId, runtimeKind: 'pi-ai' }`
- `runtime_invocation_failed` — emitted on failure, with payload `{ runId, runtimeKind: 'pi-ai', errorCategory, errorMessage }`

Use the same event shape as OpenClawCliRuntimeAdapter (traceId, timestamp, sessionId, agentId, payload).

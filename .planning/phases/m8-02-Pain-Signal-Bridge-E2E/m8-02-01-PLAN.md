In PainSignalBridge.onPainDetected(), replace the bare createTask call (lines 86-93) with idempotent upsert logic:

```typescript
// Step 1: Idempotent upsert — if task already exists, reset it for re-run
const existingTask = await this.stateManager.getTask(painId);
if (existingTask) {
  // D-06/D-07: Upsert resets attempt_count so DiagnosticianRunner re-runs
  await this.stateManager.updateTask(painId, {
    status: 'pending',
    attemptCount: 0,
    lastError: null,
    resultRef: null,
  });
} else {
  await this.stateManager.createTask({
    taskId: painId,
    taskKind: 'diagnostician',
    inputRef: painId,
    status: 'pending',
    attemptCount: 0,
    maxAttempts: 3,
  });
}
```

The existing createTask path is preserved as the "create" branch of the upsert. The updateTask path handles re-running an existing task (D-07: debug re-run scenario). No SQLite UNIQUE constraint violation can occur.

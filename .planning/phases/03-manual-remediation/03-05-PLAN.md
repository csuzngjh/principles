## Step 1: Get all prefer-destructuring errors and fix mechanically

Run this to see all prefer-destructuring errors:
```bash
npm run lint 2>&1 | grep "prefer-destructuring"
```

For each prefer-destructuring error, convert property/array access to destructuring:

**Array destructuring (const x = arr[0] → const [x] = arr):**
```typescript
// Before:
const first = items[0];
const second = items[1];

// After:
const [first, second] = items;
```

**Object destructuring (const x = obj.x → const { x } = obj):**
```typescript
// Before:
const name = user.name;
const email = user.email;

// After:
const { name, email } = user;
```

**Mixed destructuring:**
```typescript
// Before:
const name = person.name;
const first = items[0];

// After:
const { name } = person;
const [first] = items;
```

## Strategy for no-explicit-any

For each `any` type, choose one of:
1. **Replace with `unknown`** if the type can be safely narrowed
2. **Add eslint-disable with reason** if `any` is unavoidable

## Process

Run to get all no-explicit-any errors:
```bash
npm run lint 2>&1 | grep "no-explicit-any" | head -80
```

For each error, read the surrounding context and apply the appropriate fix.

**Pattern 1: Callback/iterator `any` (convert to unknown):**
```typescript
// Before:
items.forEach((item: any) => { console.log(item); });

// After:
items.forEach((item: unknown) => { console.log(String(item)); });
// OR if you need specific type behavior:
// eslint-disable-next-line @typescript-eslint/no-explicit-any -- Reason: [explanation]
items.forEach((item: any) => { console.log(item); });
```

**Pattern 2: Function parameter `any` (convert to unknown or suppress):**
```typescript
// Before:
function handle(data: any) { ... }

// After (if type can be generic):
function handle(data: unknown) {
  if (typeof data === 'string') { ... }
}

// OR suppress if truly needed:
function handle(
  // eslint-disable-next-line @typescript-eslint/no-explicit-any -- Reason: [explanation]
  data: any
) { ... }
```

**Pattern 3: Type assertion `any` (suppress with reason):**
```typescript
// eslint-disable-next-line @typescript-eslint/no-explicit-any -- Reason: third-party API returns dynamic JSON
const response: any = fetchResult;
```

## Batch suppress for unavoidable cases

If a file has MANY `any` types that all have legitimate reasons, add a file-level disable at the top:
```typescript

```

Then re-enable for specific cases that can be fixed.

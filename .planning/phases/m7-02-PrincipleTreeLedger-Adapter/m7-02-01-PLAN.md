- Accept `{ stateDir: string }` options object (DI pattern from FileStorageAdapter/EvolutionReducerImpl).
- Initialize private `#stateDir: string` and `#entryMap = new Map<string, LedgerPrincipleEntry>()`.
- Add JSDoc: "This adapter maintains an in-memory idempotency map. Create once and reuse across calls within the same process lifetime. Do not create a new instance per intake call."

**`writeProbationEntry(entry: LedgerPrincipleEntry): LedgerPrincipleEntry`:**
1. Extract candidateId from `entry.sourceRef` by stripping the `candidate://` prefix using a private method `#extractCandidateId(sourceRef: string): string`. The sourceRef format is always `candidate://<candidateId>` per CONTEXT.md Decision C and candidate-intake.ts L138.
2. Idempotency check (Decision D): `const existing = this.#entryMap.get(candidateId); if (existing) return existing;`
3. Expand to LedgerPrinciple via private `#expandToLedgerPrinciple(entry, candidateId)`.
4. Call `addPrincipleToLedger(this.#stateDir, ledgerPrinciple)` inside a try-catch. On failure, throw `new CandidateIntakeError(INTAKE_ERROR_CODES.LEDGER_WRITE_FAILED, 'Failed to write principle ${entry.id} to ledger: ${String(err)}', { cause: err })`.
5. Record in Map: `this.#entryMap.set(candidateId, entry);`
6. Return `entry`.

**`existsForCandidate(candidateId: string): LedgerPrincipleEntry | null`:**
- Return `this.#entryMap.get(candidateId) ?? null`.

**`#extractCandidateId(sourceRef: string): string`:**
- Parse `candidate://` prefix. Since sourceRef format is `candidate://<candidateId>`, use `sourceRef.startsWith('candidate://')` check then return `sourceRef.slice('candidate://'.length)`. If the prefix is missing, return the sourceRef as-is (defensive fallback).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-m7-02-01 | Tampering | writeProbationEntry | mitigate | Input is already validated by TypeBox schema (`LedgerPrincipleEntrySchema`) at the intake service boundary (m7-03). Adapter receives typed input — no additional runtime validation needed. |
| T-m7-02-02 | Tampering | addPrincipleToLedger call | mitigate | `addPrincipleToLedger` → `mutateLedger` → `withLock()` provides O_EXCL atomic lock acquisition with PID-based dead process detection and stale lock cleanup. `atomicWriteFileSync` writes to `.tmp` then renames — crash during write leaves temp file, not corrupted main file. |
| T-m7-02-03 | Information Disclosure | In-memory Map | accept | Map contains only candidate IDs (opaque strings) and principle metadata. No PII, secrets, or user data. Data is process-local and garbage-collected on process exit. |
| T-m7-02-04 | Denial of Service | existsForCandidate | accept | O(1) Map lookup. No resource exhaustion risk. Map size is bounded by number of intake calls per process lifetime. |

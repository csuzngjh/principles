<task type="auto">
  <name>Task 1: Create PDTaskSpec interface and builtin task declarations</name>
  <files>packages/openclaw-plugin/src/core/pd-task-types.ts</files>
  <instructions>
Create `packages/openclaw-plugin/src/core/pd-task-types.ts` with:

1. **PDTaskSpec interface** (from architecture doc §4.3):
   - id: string (stable unique ID)
   - name: string (human-readable, becomes CronJob name)
   - description: string
   - enabled: boolean
   - version: string (schema version for reconcile)
   - schedule: { kind: 'every'; everyMs: number }
   - agentId?: string (default: "main")
   - execution: { promptTemplate: string; timeoutSeconds?: number; lightContext?: boolean; toolsAllow?: string[] }
   - delivery: { mode: 'none' | 'announce'; channel?: string; to?: string }
   - meta?: { createdAtMs?: number; lastSyncedAtMs?: number; lastSyncedJobId?: string; lastSyncStatus?: 'ok' | 'error'; lastSyncError?: string; consecutiveFailCount?: number; lastFailedAtMs?: number; autoDisabled?: boolean; autoDisabledAt?: number; autoDisabledReason?: string; executionHistory?: Array<{ runId: string; status: string; startedAt?: number; endedAt?: number; error?: string }> }

2. **PDTaskDeclaration type** — union of PDTaskSpec with additional runtime fields

3. **BUILTIN_PD_TASKS constant** — array containing empathy-optimizer task:
   - id: 'empathy-optimizer'
   - name: 'PD Empathy Optimizer'
   - description: 'Analyzes recent user messages to discover new frustration expressions and optimize keyword weights.'
   - enabled: true
   - version: '1.0.0'
   - schedule: { kind: 'every', everyMs: 6 * 60 * 60 * 1000 } (6 hours)
   - agentId: 'main'
   - execution: { promptTemplate: 'empathy-optimizer', timeoutSeconds: 120, lightContext: true, toolsAllow: ['read_file', 'write_file', 'search_file_content'] }
   - delivery: { mode: 'none' }

Follow the pattern from empathy-types.ts and trajectory-types.ts for interface style.
  </instructions>
</task>

<task type="auto">
  <name>Task 2: Create PDTaskStore for pd_tasks.json I/O</name>
  <files>packages/openclaw-plugin/src/core/pd-task-store.ts</files>
  <instructions>
Create `packages/openclaw-plugin/src/core/pd-task-store.ts` with:

1. **PD_TASKS_PATH** — resolves to workspace `.state/pd_tasks.json`

2. **readTasks(workspaceDir: string): PDTaskSpec[]** — reads pd_tasks.json, returns array of tasks. Returns empty array if file doesn't exist. Logs warning on parse error.

3. **writeTasks(workspaceDir: string, tasks: PDTaskSpec[]): void** — writes tasks to pd_tasks.json atomically using withLockAsync from file-lock.ts. Creates .state directory if needed. Uses temp file + rename pattern for atomic write.

4. **resolvePdTasksPath(workspaceDir: string): string** — helper to resolve pd_tasks.json path

5. **ensureStateDir(workspaceDir: string): void** — creates .state directory if needed

Follow the pattern from empathy-keyword-matcher.ts (loadKeywordStore/saveKeywordStore) and model-training-registry.ts for file I/O with locking.

Use `resolvePdPath` from `paths.js` if available, otherwise construct path as `${workspaceDir}/.state/pd_tasks.json`.
  </instructions>
</task>

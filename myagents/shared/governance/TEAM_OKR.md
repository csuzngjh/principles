# TEAM_OKR

> Updated: 2026-03-18
> Model: Shared team-level OKR with per-agent local views

## Current Goal

Turn Principles Disciple from a single overloaded cloud agent into a stable semi-autonomous team.

## Current KRs

### KR1 - Team Skeleton

Success means:

- all 5 roles exist
- each role has a clear boundary
- shared governance files exist

### KR2 - Governance Reuse

Success means:

- existing `CURRENT_FOCUS`, `memory/okr/`, `WEEK_STATE`, and `WEEK_TASKS` assets are reused
- shared governance does not replace them blindly
- team-level and local-role views are both preserved

### KR3 - Safe Collaboration

Success means:

- peer roles can hand off work in a stable way
- subagents are used only for bounded work
- no default path leads directly from signal to uncontrolled coding

### KR4 - Evidence-Based Flow

Success means:

- Issue Drafts, Proposal Drafts, Repair Tasks, and Verification Reports all exist
- team progress can be summarized from shared artifacts

## Local Role Views

The following local files remain valid and should be treated as role views, not discarded systems:

- `myagents/main/memory/okr/`
- `myagents/pm/memory/okr/`
- `myagents/resource-scout/memory/okr/`

The team layer coordinates them; it does not erase them.

# 用户故事地图

## 简介

用户故事地图（User Story Mapping）是一种可视化产品规划工具，用于理解用户旅程、组织需求优先级。

## 核心框架

### 1. 用户画像

- 目标用户是谁？
- 用户的目标是什么？
- 用户的痛点是什么？
- 用户的能力和限制？

### 2. 用户旅程骨架

从左到右排列主要活动（Backbone）：

```
发现 → 了解 → 评估 → 购买 → 使用 → 推荐
```

### 3. 故事分解

在每个活动下分解具体任务：

```
发现
├── 搜索产品
├── 看到广告
├── 朋友推荐
└── 社交媒体

了解
├── 浏览产品页
├── 查看评价
├── 观看视频
└── 咨询客服
```

### 4. 优先级切片

- **MVP（最小可行产品）**：最底层的核心故事
- **V1.0**：MVP + 高优先级功能
- **V2.0**：完整产品

## 分析清单

使用此模型时，依次检查：

- [ ] 是否识别了所有用户角色？
- [ ] 是否完整映射了用户旅程？
- [ ] 是否分解到了可执行的任务？
- [ ] 是否确定了合理的优先级？
- [ ] MVP 是否真正"最小可行"？

## 常见陷阱

1. **角色不清**：忽视不同用户群体的差异
2. **过度细节**：分解过细，失去全局视角
3. **优先级混乱**：凭感觉而非价值排序
4. **忽视反馈**：不根据用户反馈迭代
